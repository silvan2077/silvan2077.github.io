了解博客基础知识

动态网页

  • 动态博客（如 WordPress、Typecho、Drupal 等）依赖数据库（MySQL、PostgreSQL 等）和后端脚本（PHP、Node.js 等）来生成页面。访问时由服务器即时渲染生成 HTML 页面。
  • 依赖数据库和后端程序
  • 页面内容可以即时更新和交互
  • 通常带有管理后台，操作简单
  • **功能丰富：**插件、主题和扩展生态庞大，支持评论、搜索、用户系统等。
  • **管理方便：**提供可视化后台，适合不懂代码的人。
  • **动态交互：**适合需要用户互动或频繁更新的场景。
  • **性能一般：**每次访问都需要数据库和后端处理，速度比静态博客慢。
  • **安全风险较高：**数据库注入、插件漏洞、服务器攻击等风险大。
  • **维护成本高：**需要服务器、数据库环境，升级和备份相对麻烦。

腾讯云

静态网页

  • 静态博客（如 Hexo、Hugo、Jekyll 等）通过生成器预先将内容编译为纯静态HTML文件，用户访问时直接读取文件，无需数据库和后端。
  • 生成静态文件，直接部署到服务器/CDN
  • 依赖 Git 等工具管理内容
  • 通常使用 Markdown 写作
  • 访问速度快：静态文件可直接由 CDN 分发，性能优异。
  • 安全性高：无数据库和动态后台，攻击面小。
  • 部署简单：可以托管在 GitHub Pages、Vercel、Netlify 等，甚至不需要独立服务器。
  • 成本低：无需购买数据库服务器，很多托管平台免费。
  • 交互性弱：不适合复杂的评论系统、用户登录等功能。
  • 更新流程繁琐：更新内容需要重新构建、部署。
  • 学习门槛较高：需要掌握 Git、命令行和静态站点生成器。
方面 动态博客 静态博客
页面生成 实时渲染（依赖数据库、后端） 预生成（纯静态 HTML）
性能 较慢，依赖服务器性能 快速，CDN 友好
安全性 风险较大 风险较低
功能 丰富，扩展性强 功能有限
维护 需数据库和环境 维护轻松
使用门槛 低（后台管理） 相对高（命令行/Git）
适用人群 非技术用户、需要交互的站点 技术用户、注重速度和简洁的博客

静态网站生成器

  • Hexo
    • 特点：基于 Node.js，生态成熟，主题插件丰富，社区大。
    • 适合人群：新手 + 想快速搭建漂亮博客的人。
  • Hugo
    • 特点：Go 语言开发，生成速度极快，支持多语言，配置灵活。
    • 适合人群：有一定技术基础、想要高性能的人。
  • Jekyll
    • 特点：Ruby 生态，GitHub Pages 官方支持，部署无痛。
    • 适合人群：想省事、直接用 GitHub Pages 的人。
  • Astro
    • 特点：新兴框架，支持多种前端框架（React/Vue/Svelte）， 适合现代化网站。
    • 适合人群：前端开发者，想结合博客和应用功能。
  • Zola
    • 特点：Rust 编写，性能和易用性兼顾，配置直观。
    • 适合人群：喜欢简洁高效的开发体验。

安装hexo

  • 直接在git中输入npm install -g hexo-cli命令即可，如果有淘宝镜像源的错误，我的做法是切换到官方源，官方源的缺点是速度可能更慢，但是稳定
  • 安装完成后，在想安装的文件夹下打开"Git Bash"，再执行斜面的命令，hexo就会新建需要的文件：
  • 初始化项目以后，blog-demo有如下结构：

【node_modules】：依赖包
【scaffolds】：生成文章的一些模板
【source】：用来存放你的文章
【themes】：主题
【.npmignore】：发布时忽略的文件（可忽略）
【_config.landscape.yml】：主题的配置文件
【config.yml】：博客的配置文件
【package.json】：项目名称、描述、版本、运行和开发等信息

成功完成以上的步骤，此时运行hexo s(serve)命令就能在http://localhost:4000/预览博客了。

下载主题

  • 在hexo的官网上选择一个喜欢的主题，点击进去后会有github的地址，直接放到themes文件夹下即可。
  • 此处采用https://github.com/Siricee/hexo-theme-Chic这个主题

将主题修改为hexo-theme-Chic-master(和主题文件名一致)

修改之后再执行hexo g,然后hexo s，就能在本地看到修改主题后的结果了。

修改配置文件

在下载的主题文件下有一个_config.yml文件，这个是主题的配置文件，打开后可以看到很多配置项，可以根据自己的需求进行修改。

安装hexo-deployer-git

hexo-deployer-git 是 Hexo 博客的核心部署插件，其核心作用是 将本地生成的 Hexo 静态博客文件（HTML、CSS、JS 等）通过 Git 协议自动部署到远程 Git 仓库（最常见的场景是部署到 GitHub Pages、GitLab Pages 等平台）。

修改配置文件

  • 打开根目录的_config.yaml文件，在文件的末尾配置上
repo的不同对应着不同的部署方式:http和ssh

我用的是ssh方式，所以不能用https的地址，否则会报错。

部署到github

  • 执行以下命令部署文件
  • hexo clean：删除之前生成的文件，可以用hexo cl缩写。若未生成过静态文件，可忽略此命令。
  • hexo generate：生成静态文章，可以用hexo g缩写
  • hexo deploy：部署文章，可以用hexo d缩写
  • 出现deploy done.就表示部署成功了,此时就能打开浏览器访问了
